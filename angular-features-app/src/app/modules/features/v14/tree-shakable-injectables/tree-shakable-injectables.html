Angular 14: Tree-Shakable Injectables

How it works

  • Use providedIn: 'root' (or any module) in your services.
  • Angular's build process excludes unused services, reducing final bundle size.
  • Services are "tree-shakable" and won't be bundled if not injected anywhere.

Benefits

  • Smaller application bundles
  • Improved startup performance
  • Cleaner dependency injection management

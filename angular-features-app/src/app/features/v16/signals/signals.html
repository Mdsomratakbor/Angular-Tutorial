Angular 16 Signals (Developer Preview)

Angular 16 introduced Signals as a developer preview feature to enable fine-grained reactivity. Signals let you store reactive state and automatically update consumers when the value changes.

Key APIs

  • signal(initialValue) – Creates a writable signal.
  • computed(fn) – Creates a derived signal.
  • effect(fn) – Runs when dependencies change.
  • set(value) – Sets a new value.
  • update(fn) – Updates based on current value.
  • mutate(fn) – Mutates the value in place.

Advantages

  • No need to manage subscriptions.
  • Less boilerplate compared to RxJS BehaviorSubject.
  • More predictable change detection.

Limitations

  • Still in developer preview in Angular 16.
  • Integration with some Angular APIs was added later (Angular 17+).
